Spring Boot Deployment Guide

AWS ECS with Fargate

Step-by-step flow from local Spring Boot app to a public URL on AWS using Docker, ECR, ECS, and Fargate.

Spring Boot app -> Docker image -> Push image to Amazon ECR -> Create ECS Task Definition -> Create ECS Cluster -> Create ECS Service using Fargate -> Access app using Load Balancer URL
ECR ECS Fargate ALB
Before You Start

Why We Need ECR

Local Docker images cannot be pulled directly by ECS. ECS tasks need a registry image URI, and Amazon ECR is the usual registry for AWS deployments.

Deployment Pattern

Build jar locally -> build Docker image locally -> push image to ECR -> deploy ECS task/service from ECR image URI.

Keep account ID and region consistent across all commands and console settings.
Step 1

Build the Spring Boot Jar

From your project folder, package the app:

mvn clean package

Expected artifact:

target/ecs-fargate-demo-0.0.1-SNAPSHOT.jar
Step 2

Build Local Docker Image

Build image from your Dockerfile:

docker build -t ecs-fargate-demo .

Local image name:

ecs-fargate-demo:latest
Step 3

Create Amazon ECR Repository

Set region and repo name (example: Mumbai):

$REGION="ap-south-1"
$REPO_NAME="ecs-fargate-demo"

Create repository:

aws ecr create-repository --repository-name ecs-fargate-demo --region ap-south-1

Get AWS account ID:

aws sts get-caller-identity --query Account --output text

ECR image URI format:

123456789012.dkr.ecr.ap-south-1.amazonaws.com/ecs-fargate-demo
Step 4

Login Docker to ECR

Authenticate Docker client with ECR:

aws ecr get-login-password --region ap-south-1 | docker login --username AWS --password-stdin 123456789012.dkr.ecr.ap-south-1.amazonaws.com
Replace 123456789012 with your actual AWS account ID.
Step 5 and Step 6

Tag and Push Image to ECR

Tag local image using ECR URI:

docker tag ecs-fargate-demo:latest 123456789012.dkr.ecr.ap-south-1.amazonaws.com/ecs-fargate-demo:latest

Push image:

docker push 123456789012.dkr.ecr.ap-south-1.amazonaws.com/ecs-fargate-demo:latest

Your app image is now available for ECS tasks.

Step 7

Create ECS Cluster

  1. Go to Amazon ECS in AWS Console.
  2. Open Clusters and click Create cluster.
  3. Cluster name: ecs-fargate-demo-cluster.
  4. Infrastructure: choose AWS Fargate.
  5. Create cluster.

Step 9

Create ECS Service (Fargate)

Service Basics

  1. Cluster: ecs-fargate-demo-cluster
  2. Launch type: Fargate
  3. Task definition: ecs-fargate-demo-task
  4. Service name: ecs-fargate-demo-service
  5. Desired tasks: 1

Networking

  1. VPC: default VPC for demo
  2. Subnets: select 2 public subnets
  3. Public IP: Enabled (simple demo path)
  4. Security groups: allow traffic as needed
Accessing the App

Simple Demo vs Recommended Setup

Simple Demo (Public Task IP)

  1. Open ECS service -> running task.
  2. Copy the task public IP.
  3. Open http://PUBLIC_IP:8080/api/hello
http://13.232.10.25:8080/api/hello

Recommended (Application Load Balancer)

  1. LB type: Application Load Balancer
  2. Listener: HTTP 80
  3. Target group: HTTP 8080
  4. Health check path: /actuator/health
  5. ECS service must be attached to this target group
http://ecs-demo-alb-123456.ap-south-1.elb.amazonaws.com/api/hello
Important Missing Bridge

How ALB Finds the ECS Task

The target group does not know ECS task IPs by itself. ECS registers task private IPs only when the ECS service is connected to the target group.

ECS Service starts a Fargate task -> Task receives a private IP, for example 10.0.2.55 -> ECS service registers 10.0.2.55:8080 in the target group -> ALB forwards http://ALB-DNS/api/hello to 10.0.2.55:8080/api/hello
For Fargate, the target group target type must be IP, because Fargate tasks are registered by private IP address.
ECS Service Load Balancing

Attach ECS Service to ALB

Console Steps

  1. Open ECS -> Cluster -> Services.
  2. Create a new service if the current service was created without ALB.
  3. Use the same task definition.
  4. In Load balancing, choose Application Load Balancer.
  5. Select ALB listener HTTP:80.
  6. Select the target group using HTTP:8080.

Container Mapping

  1. Container name: ecs-fargate-demo
  2. Container port: 8080
  3. Health check grace period: 60 seconds
  4. After service starts, check EC2 -> Target Groups -> Targets.
  5. Wait until the registered task IP is Healthy.
Without this ECS service load balancing step, the ALB and target group can exist but still return 503 because no healthy ECS task is registered.
Security and Core Terms

Security Group Rules + ECS Concepts

Security Group Rules

  1. Without ALB: allow inbound 8080 from your IP.
  2. With ALB: ALB SG allows inbound 80 from internet.
  3. With ALB: ECS task SG allows inbound 8080 only from ALB SG.

ECS Terms

  1. ECR: stores Docker images.
  2. Cluster: logical environment where tasks run.
  3. Task definition: image + CPU/memory + port blueprint.
  4. Task: one running container copy.
  5. Service: keeps desired number of tasks running.
  6. Fargate: serverless compute for containers.
